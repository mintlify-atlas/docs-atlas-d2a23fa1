## Get Activity Logs

<CodeGroup>
```bash GET /activity-logs
curl -X GET "http://localhost:8080/api/v1/activity-logs?page=1&limit=10&entity_type=ORDER&action_type=CREATE" \
  -H "Authorization: Bearer YOUR_TOKEN"
```
</CodeGroup>

Get a list of activity logs with filtering and pagination. This endpoint is restricted to administrators only.

### Query Parameters

<ParamField query="page" type="integer" default="1">
  Page number for pagination
</ParamField>

<ParamField query="limit" type="integer" default="10">
  Number of items per page (max 100)
</ParamField>

<ParamField query="search" type="string">
  Search term to filter logs
</ParamField>

<ParamField query="start_date" type="string">
  Start date filter in YYYY-MM-DD format
</ParamField>

<ParamField query="end_date" type="string">
  End date filter in YYYY-MM-DD format
</ParamField>

<ParamField query="user_id" type="string">
  Filter by user UUID
</ParamField>

<ParamField query="entity_type" type="string">
  Filter by entity type:
  - `PRODUCT`
  - `CATEGORY`
  - `PROMOTION`
  - `ORDER`
  - `USER`
  - `SETTINGS`
  - `SHIFT`
  - `PAYMENT_METHOD`
  - `CANCELLATION_REASON`
</ParamField>

<ParamField query="action_type" type="string">
  Filter by action type:
  - `CREATE`
  - `UPDATE`
  - `DELETE`
  - `CANCEL`
  - `APPLY_PROMOTION`
  - `PROCESS_PAYMENT`
  - `REGISTER`
  - `UPDATE_PASSWORD`
  - `UPDATE_AVATAR`
  - `LOGIN_SUCCESS`
  - `LOGIN_FAILED`
  - `RESTORE`
</ParamField>

### Response

<ResponseField name="data" type="object">
  <Expandable title="ActivityLogListResponse">
    <ResponseField name="logs" type="array" required>
      Array of activity log entries
      
      <Expandable title="ActivityLogResponse">
        <ResponseField name="id" type="string" required>
          Log entry UUID
        </ResponseField>
        
        <ResponseField name="user_id" type="string" required>
          User UUID who performed the action
        </ResponseField>
        
        <ResponseField name="user_name" type="string" required>
          Username of the user who performed the action
        </ResponseField>
        
        <ResponseField name="action_type" type="string" required>
          Type of action performed
        </ResponseField>
        
        <ResponseField name="entity_type" type="string" required>
          Type of entity affected
        </ResponseField>
        
        <ResponseField name="entity_id" type="string" required>
          ID of the affected entity
        </ResponseField>
        
        <ResponseField name="details" type="object" required>
          Additional details about the action (JSON object)
        </ResponseField>
        
        <ResponseField name="created_at" type="string" required>
          Timestamp when the action occurred
        </ResponseField>
      </Expandable>
    </ResponseField>
    
    <ResponseField name="total_items" type="integer" required>
      Total number of log entries
    </ResponseField>
    
    <ResponseField name="total_pages" type="integer" required>
      Total number of pages
    </ResponseField>
    
    <ResponseField name="page" type="integer" required>
      Current page number
    </ResponseField>
    
    <ResponseField name="limit" type="integer" required>
      Items per page
    </ResponseField>
  </Expandable>
</ResponseField>

### Roles

Accessible by: `admin` only

<Note>
Activity logs are automatically created by the system when users perform actions. This endpoint is read-only and provides audit trail functionality for administrators.
</Note>
